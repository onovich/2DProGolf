# 2D Pro Golf RPG 化策划文档

## 1. 项目定位

项目暂定名：2D Pro Golf: Another Fairway

产品定位：以 2D 程序化高尔夫为核心玩法、以异世界穿越叙事为驱动的轻 RPG 单机游戏。

核心卖点：

1. 用真实但可理解的高尔夫物理，提供“判断地形、风向、落点”的策略快感。
2. 用碎碎念风格的 NPC 对话和上班族异世界群像，提供强记忆点叙事体验。
3. 用城市地形特色、赛事规则变体和对手风格差异，持续扩展玩法内容。
4. 用“从普通社畜到世界冠军，再被神灵一杆打回地球”的荒诞目标，形成完整闭环。

这个方向是成立的，但建议定义为“轻 RPG 高尔夫冒险”而不是“纯 RPG”。原因是当前项目最有价值的部分仍然是击球、风、坡度、地形生成。如果后续把成长、剧情、收集做得过重，反而会稀释已经存在的玩法辨识度。

## 2. 设计原则

1. 高尔夫是第一主循环，剧情是推进器，RPG 是包装层。
2. 玩家成长应提升理解力、容错率和表达方式，不应直接让物理玩法失效。
3. 每个城市都必须提供新的地形问题、规则问题或对手问题，不能只换美术皮肤。
4. 对话必须有角色感，但不做大段严肃史诗，整体保持轻荒诞、碎碎念、带一点现实疲惫感。
5. 体验上优先“可读、可学、可复盘”，再追求“拟真、困难、深度”。

## 3. 当前原型的问题判断

基于现有版本，当前产品更像“程序化高尔夫原型”，尚未形成完整内容产品。主要问题如下：

1. 风力影响过大，逆风场景会让玩家感到击球结果失真，甚至出现“球几乎无法向目标推进”的挫败感。
2. 球洞内容主要来自随机地形，没有明确的关卡主题、节奏设计和挑战曲线。
3. 玩家缺少长期目标，当前只有打一洞、进下一洞，没有章节、成长、角色关系和赛事线。
4. UI 只提供基础状态，没有坡度辅助、落点理解、地形提示和赛事上下文。
5. 参数分散在引擎内部，策划难以持续调优风力、洞型、障碍密度和球杆手感。

## 4. 核心改造方向

### 4.1 风力系统改造

风力必须从“经常抢走控制权的系统”改成“迫使玩家修正决策的系统”。

调整原则：

1. 风力影响保留，但不能覆盖击球主方向。
2. 逆风应该缩短距离，而不是频繁产生“几乎原地挣扎”的违和结果。
3. 风力读法必须直观，玩家要能理解为什么偏、偏了多少、下次该怎么修。

建议调整方案：

1. 全局风力强度先下调 35% 到 50%，优先改善初版体验。
2. 风力只显著影响滞空阶段，对地滚球和低弹道球的干扰进一步降低。
3. 为不同球杆设定受风系数：木杆最低，铁杆中等，挖起杆最高，推杆为零。
4. 对逆风加入“最小推进保障”，即逆风最多削减一部分前进速度，但不应该经常把主方向速度压到几乎消失。
5. 把风拆成“稳定风”与“阵风事件”两层。常规比赛默认稳定风，少数城市或高难洞再引入阵风。
6. 在瞄准阶段增加一个简化的风偏预估标记，让玩家能理解大致偏移趋势。

数值建议：

1. 当前版本先把基础风速上限压低到易于学习的区间，再通过城市和赛事逐步抬升。
2. 新手阶段尽量避免逆风加大上坡再叠加长洞的三重惩罚组合。
3. 挖起杆要保留明显受风特性，但不能因为风而失去“精准停球”这个定位。

### 4.2 产品形态改造

产品从“无限程序化打洞”升级为“有主线的城市巡回高尔夫冒险”。

一句话体验：

玩家作为误入高尔夫异世界的普通上班族，在精灵向导的带领下学习规则、参加比赛、结识其他穿越者，最终赢得世界冠军，并在通天塔被神灵当作高尔夫球击回地球。

## 5. 世界观与叙事设定

### 5.1 故事起点

主角是一名普通上班族，在一次车祸后失去意识，醒来时发现自己来到一个以高尔夫为最高秩序的异世界。这里的居民并不都是原住民，很多关键人物也都是从地球穿越而来，只是来自不同职业、不同年龄、不同社会位置。

他们有人已经接受命运，有人仍然执着于回家，有人把高尔夫当工作，有人把高尔夫当宗教，有人嘴上说不在乎、实际上非常在乎排名。

### 5.2 引导角色

精灵向导是主角的长期陪伴角色，功能如下：

1. 前期担任教学者，负责解释规则、球杆、风、地形与赛事系统。
2. 中期担任吐槽和复盘者，对主角每场比赛给出简短评价。
3. 后期担任主线推进者，引导玩家走向世界冠军赛和神灵之所。
4. 情绪上既是功能型导航，也是作品的节奏调节器。

精灵的文本风格建议：嘴碎、轻快、偶尔毒舌，但不能过度喧宾夺主。

### 5.3 NPC 群像原则

1. NPC 全部是从地球来到异世界的人。
2. 角色来源涵盖上班族、老板、自由职业者、艺术家、基层劳动者、科研人员、学生、退役运动员等。
3. 对话风格偏碎碎念，不强调宏大叙事，而强调“具体的人在抱怨具体的生活”。
4. NPC 的打球风格与其性格不必一致，这种反差本身就是乐趣来源。
5. 球场上的部分观众可交互，对话更短、更即时、更像路人闲聊。

### 5.4 结局表达

成为世界冠军后，主角抵达通天塔。神灵并不以传统史诗方式出现，而是以一种近乎游戏规则本体的形式出现。最终镜头停留在主角被置于发球点、天空展开、击球声响起、画面黑屏，字幕呈现“回到地球与否”的留白式余韵，不直接展示现实落地过程。

## 6. 核心玩法循环

### 6.1 大循环

1. 在城镇中接收赛事、对话、训练、补给和剧情。
2. 进入球场，完成练习、单人赛、多人赛或特殊规则赛。
3. 根据成绩获得排名点数、货币、装备材料、声望和剧情推进。
4. 解锁新城市、新规则、新球杆改造、新 NPC 支线。
5. 最终进入世界冠军赛和通天塔终章。

### 6.2 单场循环

1. 赛前查看球场情报，包括地形、风况、赛事规则和对手特点。
2. 击球过程中做落点、球杆、风向和坡度判断。
3. 赛间触发 NPC 吐槽、看客对话或精灵点评。
4. 赛后进入成绩结算、剧情反馈和下一目标指引。

### 6.3 成长循环

1. 学会新规则。
2. 获得新辅助能力或训练结果。
3. 挑战更复杂球场。
4. 认识更强对手。
5. 回到更高压力赛事中验证自己。

## 7. RPG 系统设计

### 7.1 角色成长

不建议做传统等级制加攻击力，而建议做“能力理解型成长”。

成长方向建议：

1. 读风能力：解锁更清晰的风偏提示、风况描述词、阵风预警。
2. 地形理解：显示更明确的坡度辅助、果岭流线、地表标签。
3. 稳定心态：降低极端失误惩罚，提升糟糕落点后的容错能力。
4. 球杆熟练度：提升某类球杆的手感窗口和击球反馈质量。
5. 社交能力：解锁特殊对话、额外赛事、城市传闻和隐藏练习场。

### 7.2 装备系统

装备不应改成粗暴加数值，而应做成轻策略选择。

装备层建议：

1. 球杆部件：影响弹道、受风系数、停球表现、稳定性。
2. 球种：影响旋转、落地反弹、特殊地形表现。
3. 饰品：影响赛后奖励、对话触发率、训练效率等外围收益。

### 7.3 赛事与排名系统

赛事类型建议：

1. 练习赛：规则简化，用于教学和试杆。
2. 正式巡回赛：标准杆赛，逐步构成主线主体。
3. 对抗赛：与单一 NPC 对手比拼，强调角色表现。
4. 多人赛：多名 AI 同场计分，强调名次与节奏压力。
5. 特殊规则赛：限定球杆、强风日、连洞挑战、限杆数通关。

### 7.4 NPC 关系系统

NPC 关系系统不需要做复杂好感模拟，但应有轻量推进。

实现目标：

1. 让玩家感觉角色之间关系在变化。
2. 让对话和赛事内容互相反哺。
3. 让重复造访城市有意义。

表现方式：

1. 比赛前后短对话。
2. 达成特定成绩后触发额外片段。
3. 某些 NPC 在不同城市再次出现，形成漂流者共同体的感觉。

## 8. 城市与章节结构

建议采用 5 座城市加终章通天塔的结构。

### 8.1 第一章：雾岸练习镇

定位：教学区与情绪缓冲区。

特点：

1. 低风、短洞、宽球道。
2. 教基础规则、球杆分工、简单沙坑和轻微坡度。
3. NPC 以“刚穿越不久的人”为主，对世界仍有强烈困惑感。

### 8.2 第二章：砂页城

定位：障碍思维入门。

特点：

1. 沙坑密集，落点规划重要。
2. 开始引入明显侧风。
3. 赛事会强调“保守打”与“激进切线”的选择。

### 8.3 第三章：雨幕沼都

定位：水域与罚杆压力。

特点：

1. 水域多，落水惩罚频繁出现。
2. 草地湿软，滚动距离偏短。
3. 对手更擅长保守推进，逼迫玩家理解风险管理。

### 8.4 第四章：崖线钢都

定位：高低差与路线阅读。

特点：

1. 起伏强，高低差大。
2. 城市场地带有人造结构感，可加入更窄的球道与平台果岭。
3. 规则开始更复杂，包含连续洞赛事和多人排名赛。

### 8.5 第五章：白昼冠城

定位：世界级赛事舞台。

特点：

1. 综合考验风、水、坡、障碍与心理压力。
2. 顶级选手云集，多个旧角色回归。
3. 世界冠军赛在此完成。

### 8.6 终章：通天塔

定位：象征性终局关卡。

特点：

1. 洞数不多，但每洞都强调概念性与仪式感。
2. 精灵和重要 NPC 的文本回收集中发生在这里。
3. 最终击球成为叙事表达而非普通比赛。

## 9. 对手设计

对手的重点不在“更高数值”，而在“更有辨识度的出杆选择”。

AI 打法类型建议：

1. 稳健型：优先避险，少失误，但抓鸟能力一般。
2. 激进型：敢切线、敢赌风，容易打出高光也容易爆洞。
3. 短杆型：长杆一般，但果岭附近极强。
4. 推进型：偏爱低弹道长推进，对风适应性强。
5. 情绪型：领先和落后时策略明显变化，表现更像人。

这些打法要与人物设定形成反差，而不是简单映射。例如，话很多的艺术家可以是极度稳健的保守型；沉默寡言的老板可以是鲁莽的激进型。

## 10. 文本与叙事风格

文本关键词：碎碎念、现实感、荒诞、轻疲惫、带一点冷幽默。

文本规则：

1. 避免纯说明书式对白。
2. 避免过度网络段子化，保持角色差异。
3. 避免把每个 NPC 都写成强设定角色，更像“真实的人偶尔多说了几句”。
4. 精灵、选手、观众、工作人员要有明显话语密度差异。

## 11. 内容制作原则

### 11.1 关卡内容

程序化不能完全自由生长，必须引入“模板 + 变体”模式。

建议做法：

1. 先定义洞型模板，例如短直洞、长直洞、左狗腿、右狗腿、隔水攻果岭、平台果岭、连续沙坑压迫洞。
2. 每个城市使用不同模板池和参数权重。
3. 再在模板基础上做程序化扰动，保证既有变化又有设计感。

### 11.2 内容产能

首个可玩版本建议目标：

1. 1 个完整序章。
2. 2 座完整城市。
3. 20 到 30 个 NPC 原型。
4. 5 类对手打法。
5. 30 到 40 组可复用短对白。
6. 3 种赛事规则变体。

完整正式版目标：

1. 5 座城市加终章。
2. 60 到 100 个具名 NPC 与路人话池。
3. 8 到 10 类赛事规则。
4. 50 个以上洞型模板变体。

## 12. 系统需求清单

### 12.1 体验与平衡优先项

1. 下调风力强度并重做受风模型。
2. 增加风偏预估和地形可读性辅助。
3. 将生成参数、赛事参数、城市参数、球杆参数全部配置化。
4. 增加固定随机种子与测试球场，用于稳定调参。

### 12.2 内容框架优先项

1. 主线章节结构。
2. 世界地图选城。
3. 赛事结算界面。
4. NPC 对话系统。
5. AI 对手计分系统。

### 12.3 表现优先项

1. 城市主题视觉差异。
2. 赛前赛后过场。
3. 精灵评论与赛事播报。
4. 基础音效与环境氛围。

## 13. 开发阶段计划

以下计划按“先把原型做成产品，再把产品做成 RPG”推进。

### 阶段 0：原型校正与可配置化

目标：先把当前游戏调成稳定、可学、可持续改的状态。

交付内容：

1. 风力系统重平衡。
2. 风力与球杆受风系数配置表。
3. 洞型生成参数配置表。
4. 地形、球杆、规则调试面板。
5. 固定测试洞与回放校验流程。

完成标准：

1. 新手在低级球洞中不会频繁遇到“逆风几乎打不动”的体验。
2. 主要球杆定位清晰。
3. 关卡难度不再随洞数无限膨胀。

### 阶段 1：叙事外壳与教学序章

目标：让玩家感受到“这是一个有故事目标的游戏”。

交付内容：

1. 序章剧情。
2. 精灵向导角色。
3. 第一座城市与教学赛。
4. 对话框、角色立绘占位、赛后评价界面。
5. 基础任务与章节推进。

完成标准：

1. 玩家可从开场一路完成教程并理解长期目标。
2. 教学不再只是系统说明，而是与剧情推进绑定。

### 阶段 2：城市与赛事框架

目标：建立可扩展的主循环。

交付内容：

1. 世界地图与选城逻辑。
2. 2 到 3 座城市内容。
3. 标准赛、对抗赛、多人赛基础框架。
4. AI 对手回合与排行榜系统。
5. 城市专属地形参数与规则变体。

完成标准：

1. 玩家能够在城市之间移动并体验明显不同的球场问题。
2. NPC 与赛事内容开始形成联动。

### 阶段 3：RPG 深化与主线扩展

目标：把系统和内容真正织成一个世界。

交付内容：

1. 角色成长系统。
2. 装备与训练系统。
3. NPC 关系推进与支线。
4. 第四、第五城市与世界冠军赛。
5. 过场、播报、演出强化。

完成标准：

1. 玩家除了成绩，还会为了剧情、人物和成长持续推进。
2. 比赛外内容不再只是菜单，而有明确存在感。

### 阶段 4：终章与整体打磨

目标：让作品具备完整的通关体验和主题收束。

交付内容：

1. 通天塔终章。
2. 结局演出与字幕收束。
3. 全流程平衡修正。
4. 文本润色与内容补量。
5. 新手体验与后期节奏复查。

完成标准：

1. 游戏可以完整通关。
2. 从教学到终局的体验曲线连续。
3. 主线主题与玩法主题能互相支撑。

## 14. 版本优先级建议

### P0 必做

1. 风力重做。
2. 参数配置化。
3. 序章教学。
4. 精灵向导。
5. 第一城市。
6. 基础对话系统。
7. 基础赛事结算。

### P1 应做

1. 世界地图。
2. 第二、第三城市。
3. AI 对手与多人排名赛。
4. 球场观众互动。
5. 基础成长与训练。

### P2 可延后

1. 装备深度系统。
2. 更完整的支线结构。
3. 更复杂的结局分支。
4. 更大量的随机路人文本。

## 15. 风险与控制建议

1. 最大风险是范围膨胀。必须先做轻 RPG 骨架，不要一开始就做大规模分支剧情。
2. 第二风险是剧情压过玩法。任何新增系统都要回答“它是否强化了击球体验”。
3. 第三风险是程序化关卡缺少可控性。必须尽早转向模板化生成。
4. 第四风险是文本产能不足。应优先建立角色模板和对话规则，再批量写内容。
5. 第五风险是 AI 与赛事系统复杂度过高。建议先做可解释、可回放的简化对手模型。

## 16. 结论

这个项目非常适合往“高尔夫异世界轻 RPG”方向升级，因为它已经拥有一个少见的核心玩法基础：风、坡度、程序化地形和球杆差异。但要成功，关键不是简单往外叠剧情和 NPC，而是先把现有原型的体验问题解决，尤其是风力失衡与参数难调的问题，再用城市、对手、赛事和群像叙事把玩法包起来。

建议的执行顺序只有一句话：先修手感和参数，再做序章和城市，再扩主线和 NPC，最后完成冠军赛与终章。
