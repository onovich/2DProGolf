# 2D Pro Golf 未来两周执行清单

## 1. 计划目标

这份两周计划不是为了继续无上限扩新内容，而是为了把当前已经闭合的主线版本，打磨成一个更稳定、更容易验收、也更便于继续维护的版本。

两周内的工作目标只有三个：

1. 把终章和通关页做成真正可稳定验收的收尾层。
2. 降低后续继续开发时的回归成本。
3. 在不稀释高尔夫主玩法的前提下，补齐最关键的可读性和体验缺口。

## 2. 总体优先级

### 第一优先级

终章与通关页的真实手动验收、移动端体验和表现层修整。

### 第二优先级

围绕 gameEngine.js 和 phase 流转做小规模维护性整理。

### 第三优先级

补更多低成本调试入口与中段验证手段，降低回归成本。

## 3. 第 1 周

### 任务 A：终章与通关页手动验收

目标：把当前已经存在的 debug 入口变成真正可执行的验收流程。

执行项：

1. 固定记录 ending 和 tour-summary 两个预设页面的桌面端检查项。
2. 固定记录移动端视口下的检查项，包括滚动、按钮顺序、信息密度和阅读顺序。
3. 把发现的问题整理成一份短表，而不是零散聊天记录。

验收标准：

1. 终章页与通关页各自有一份清晰的手动检查清单。
2. 至少完成一轮桌面端与移动端实测。
3. 所有问题都有“是否本周处理”的判断。

### 任务 B：通关页阅读顺序优化

目标：让通关页的信息组织更像赛后总结，而不是堆很多卡片。

执行项：

1. 检查移动端下“打法画像、阶段成绩、完赛记录、巡回心得”的阅读顺序。
2. 视需要微调卡片顺序、间距和滚动体验。
3. 保证重要信息先于次级信息被看到。

验收标准：

1. 移动端不再出现明显的信息拥挤或阅读跳跃。
2. 通关页顶部 1 到 2 屏内能清楚看到玩家最关心的结论层。

### 任务 C：终章页表现层微调

目标：让 ending page 和 tour summary 在视觉和情绪上拉得更开。

执行项：

1. 检查终章页标题、标签、正文和按钮间距是否足够像“结尾页”。
2. 视需要增加更克制的尾声强调，而不是继续堆叙述。
3. 确认 ending -> tour-summary 的按钮转场阅读是自然的。

验收标准：

1. 玩家能明显感知“结尾页”和“通关总览页”是两个不同阶段。
2. 从 ending 进入 summary 的动机和文案都成立。

## 4. 第 2 周

### 任务 D：维护性整理 gameEngine.js

目标：控制继续膨胀的风险，但不做大拆大改。

执行项：

1. 只围绕 phase、summary/ending/tour-summary 相关逻辑做小范围整理。
2. 收敛重复的显示/隐藏面板操作和 debug preset 相关逻辑。
3. 如有必要，把明显独立的辅助计算再抽成局部 helper，而不是开大规模模块化重构。

验收标准：

1. 新增逻辑块比现在更容易定位。
2. 没有引入流程回归。
3. 构建和关键调试预设仍然可用。

### 任务 E：补中段调试预设

目标：让后续中盘内容也能低成本验收。

执行项：

1. 评估是否为白昼冠城或世界冠军赛补 1 到 2 个 query 级别的 debug preset。
2. 保持和 ending / tour-summary 预设同一风格，不做临时脚本堆积。

验收标准：

1. 至少有一个中段关键节点可以直接打开。
2. 新预设不会污染正常玩家流程。

### 任务 F：文档与验证流程闭环

目标：把“怎么验证”和“当前优先级是什么”继续写清楚。

执行项：

1. 把本周实际手动验收项反写回文档。
2. 如果 debug 入口或验收方式新增，补进 README 或 handoff guide。
3. 把已经确认不再优先的方向标清楚，避免回到盲目扩内容模式。

验收标准：

1. 新接手的人能在 10 到 15 分钟内知道怎么复现终章和通关页。
2. 文档与实际可执行流程一致。

## 5. 两周内不建议插队的事项

1. 再开新的城市主线。
2. 引入重装备系统或传统等级制。
3. 做大规模引擎拆分重构。
4. 补大量新 NPC 支线而不先做验收和维护性整理。

## 6. 建议的每周固定验证动作

1. npm run build
2. ?debug=ending 页面检查
3. ?debug=tour-summary 页面检查
4. 至少一轮移动端视口检查

## 7. 两周后理想成果

如果这两周按这个计划推进，理想结果应该是：

1. 终章与通关页已经不只是“功能上存在”，而是“可以稳定验收”。
2. gameEngine.js 没有继续明显恶化。
3. 后续继续推进时，回归成本比现在更低。
4. 项目进入“可持续打磨”的阶段，而不是继续靠冲刺堆功能。
