项目交接文档：2D Pro Golf

1. 项目基本信息

项目英文名: 2D Pro Golf - Natural Generation (或简称 2D Pro Golf)

项目类型: 2D 物理模拟/休闲体育网页游戏

核心技术栈: HTML5 Canvas (原生 2D API), JavaScript (ES6), Tailwind CSS

依赖库: 无外部 JS 库依赖，仅通过 CDN 引入 Tailwind CSS 进行 UI 样式排版。

2. 需求设计文档 (PRD 摘要)

2.1 游戏概述

一款追求真实物理反馈的 2D 俯视角高尔夫游戏。游戏不使用预设地图，而是通过程序化生成 (Procedural Generation) 每次提供独一无二的球洞。游戏旨在通过简单的拖拽操作，结合地形高低起伏、风向和不同球杆的特性，模拟真实高尔夫的策略感。

2.2 核心机制

程序化地形 (Procedural Terrain): 使用多重分形噪音 (fBm) 生成连续的自然地形高度，并通过距离场约束动态雕刻出球道、果岭、沙坑和水域。地形具有等高线视觉反馈。

真实高尔夫物理:

摩擦力: 不同地形（果岭、球道、长草、沙坑）具有不同的摩擦系数。

坡度重力: 物理引擎根据地形梯度（法线）计算重力在 XY 平面的分量，球会自动向低处滚动。

3D 滞空模拟: 虽然画面是 2D 的，但引擎包含 Z 轴（高度）计算。球在空中时不受地面摩擦和坡度影响，但会受到强风偏移，并有动态阴影缩放效果。

球杆系统:

木杆 (1W): 力量最大，弹道低，滚动距离远。

铁杆 (7I): 均衡，适合中距离。

挖起杆 (PW): 弹道极高，受风影响大，落地后几乎不滚动（急停）。

推杆 (PT): 纯地滚，无视风力，受地形坡度影响极大。只能在果岭附近使用（或自动切换）。

交互分离: 区分“瞄准击球”与“观察地形”。点击球附近拖拽为瞄准；点击空白处拖拽为平移相机；双击恢复相机跟随球。

环境因素: 随机生成的风力，影响空中球的飞行轨迹。

3. 开发文档 (技术实现细节)

3.1 文件结构

当前项目为单文件应用：golf_game.html。包含了所有的 HTML 结构、CSS 样式（行内及 Tailwind）和核心 JavaScript 逻辑。

3.2 核心模块说明

1. 核心数学与噪音工具:

Vec2: 基础的二维向量类，用于位置、速度、力等计算。

smoothstep / distPointToSegment: 用于计算地形边缘的平滑过渡。

SimplexNoise / fBm: 实现分形布朗运动，用于生成自然随机的高度图 (Heightmap)。

2. 游戏配置 (CONSTANTS / TERRAIN / CLUBS):

定义了物理常量（重力、缩放比例）、地形属性（摩擦系数、颜色基底）和球杆属性（最大力量、仰角系数、空气阻力）。

3. 地形算法 (getTerrainAt, getGradient):

关键函数: getTerrainAt(x, y) 接受世界坐标系，返回该点的地形类型（果岭、水域等）和海拔高度。该算法通过混合基础高度图、球洞距离、路径距离等“雕刻”出合理的高尔夫球场。

getGradient(x, y) 计算某点的地形斜率（法线），用于物理计算。

4. 渲染初始化 (generateCourse):

双 Canvas 策略: 性能优化的关键。使用离屏 Canvas (#bg-canvas) 预渲染并缓存复杂的像素级地形（包括等高线光影），每一帧主循环只负责将缓存内容贴图，极大地降低了 CPU 负担。

5. 交互与输入解耦:

统一处理 Mouse 和 Touch 事件。通过判断点击点距球的距离，区分 isAiming（瞄准）和 isPanning（相机平移）状态。

6. 物理循环与渲染循环 (updatePhysics, renderLoop):

物理更新区分“空中状态” (inAir) 和“地面滚动状态”。空中受 Z 轴重力和风力影响；地面受坡度引力和摩擦力影响。落水、进沙坑等特殊情况在此处理。

渲染循环负责相机平滑跟随、拖拽轨迹预览线绘制、3D 高尔夫球投影及风效粒子的绘制。

4. 美术与视觉约束

由于是程序化生成的像素渲染游戏，美术约束主要集中在色彩与表现逻辑上：

色彩体系:

采用相对沉稳、真实的绿色调（果岭浅翠绿，球道草绿，长草墨绿）。

等高线非纯黑线，而是通过高度取模产生的柔和阴影，保留地形的体积感。

水域为蓝色，沙坑为沙色。

UI 风格:

采用现代、扁平化的深色毛玻璃风格 (Dark Glassmorphism)。

主要使用 Tailwind 的 gray-800, emerald-400 (点缀色), 和半透明背景。

3D 表现:

2D 画布必须维持 3D 深度感。球在空中的缩放系数与其 z 值挂钩；球的阴影需随着 z 值增大而变淡、位移并远离球体。

5. TODO (后续优化建议)

性能优化:

目前在极高分辨率或低端设备上生成地形（双层 for 循环遍历全图计算 fBm）可能会有卡顿（当前使用 loading 遮罩掩盖）。未来可考虑使用 Web Worker 将地形生成移出主线程，或尝试用 WebGL/Shaders 进行并行计算加速。

内容扩展:

多球洞连续计分制: 增加 9 洞或 18 洞的总杆数统计和排行榜。

植被系统: 在长草区随机生成一些简单的 2D 树木障碍（需处理树木对击球的遮挡和碰撞阻挡）。

坡度网格网 (Grid System): 像真实的高尔夫游戏一样，在瞄准时或者在果岭上，可以开启覆盖地表的移动网格线，帮助玩家更直观地判断坡度流向。

音效系统:

添加击球音效（木杆/铁杆声音区分）、落水声、进洞清脆声以及持续的环境风声。

6. 项目启动方式

本项目为纯前端静态文件，没有任何构建工具或后端依赖。

启动步骤:

确保电脑连接互联网（因需加载 CDN 上的 Tailwind CSS 库）。

在任意操作系统上，找到项目根目录下的 golf_game.html 文件。

双击该文件，使用现代浏览器（如 Google Chrome, Microsoft Edge, Firefox, Safari）打开即可直接运行并游玩。

(建议使用 Chrome 或 Edge 浏览器以获得最佳的 Canvas 渲染性能)
